## branch4_input_validation_fixes

### fix: прошедшая дата в напоминаниях
- Бот принимал дату в прошлом и создавал напоминание, которое никогда не срабатывало
- Добавлена проверка: `run_at > now_kyiv()`, иначе — ошибка пользователю

### fix: валидация текста напоминания
- Текст теперь не может быть пустым или длиннее 500 символов

### fix: валидация названия города (погода)
- Бот принимал многострочный мусор (в т.ч. SQL-инъекции) как название города
- Добавлена проверка regex: только буквы/цифры/пробелы/дефисы, одна строка, макс 50 символов

### fix: валидация имени в днях рождения
- Аналогичная валидация: только буквы/цифры/пробелы/дефисы, макс 50 символов

### fix: вызов бота на дуэль
- Бот принимал вызов от пользователя на дуэль с самим собой
- Добавлена проверка `opponent.id == bot_info.id`

### fix: дробное время мута в дуэли
- `/duel @user 1.5` — `int("1.5")` бросал `ValueError`, молча использовался дефолт
- Заменено на `round(float(arg))`, дробные значения корректно округляются

### fix: мут с меньшим сроком перебивал активный мут
- Проигрыш в рулетке мог снять более длинный мут от дуэли (и наоборот)
- Добавлена функция `get_active_mute_until()` в репозиторий
- Новый мут применяется только если он длиннее существующего
- Затронуто: `duel.py`, `roulette.py`

## branch5_logging

### feat: структурированное логирование
- Добавлен `app/config/logging_config.py`: единая конфигурация логов для Railway (только stdout)
- Библиотеки `aiogram`, `aiohttp`, `apscheduler`, `asyncio` переведены на уровень `WARNING` — красный шум в Railway Dashboard значительно уменьшился
- Бот пишет свои события на уровне `INFO`

### feat: глобальный обработчик ошибок с уведомлением в Telegram
- Добавлен `app/bot/error_handler.py`
- Любое необработанное исключение в хендлере → личное сообщение суперадмину с полным traceback, именем чата и пользователя
- Не требует внешних сервисов

### feat: интеграция Sentry
- Добавлен `sentry-sdk[aiohttp]` в зависимости
- Инициализируется в `main.py` только если задан `SENTRY_DSN` в переменных окружения — при отсутствии ключа Sentry просто не подключается
- Добавлен `SentryContextMiddleware`: к каждой ошибке прикрепляется `user_id`, `username`, `chat_id`, `chat_title`

### fix: утечка токена бота через Sentry
- Performance tracing (`traces_sample_rate`) логировал HTTP-запросы к Telegram API — URL содержал токен бота в открытом виде
- Отключён performance tracing (`traces_sample_rate=0`)
- Добавлен `before_send` хук `_scrub_token`: заменяет токен на `[REDACTED]` во всех исходящих событиях Sentry как страховочный слой

## branch8_reaction_author_fix

### fix: `to_user_id` всегда был NULL в таблице реакций
- Telegram Bot API не передаёт автора сообщения в событии `MessageReactionUpdated` — только `message_id`
- Из-за этого `to_user_id` хардкодом записывался как `NULL`
- Сломанное поведение: `/my_reactions` всегда показывал 0, топ «кто получает больше всех реакций» был пуст, награды **Клоун месяца** и **Любимчик** никогда не выдавались

### feat: таблица `MessageAuthor` для отслеживания авторства
- Новая лёгкая таблица `MessageAuthor (chat_id, message_id, user_id)` — только маппинг, без текста сообщений
- `RegisterMiddleware` при каждом входящем сообщении тихо сохраняет автора
- Хендлер реакций делает lookup по `message_id` и заполняет `to_user_id` корректно
- Автоматическая чистка: 1-го числа каждого месяца в 03:00 удаляются записи старше 30 дней — таблица не растёт бесконечно
- Исторические реакции (до деплоя) остаются с `NULL` — новые будут корректными
